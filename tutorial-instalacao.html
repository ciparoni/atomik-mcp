Atomik MCP

Instale em
4 passos.

Do zero ao primeiro componente no Elementor, sem nenhuma linha de código.

⏱ 5 minutos 💻 Nível iniciante

O que é o MCP do Atomik?

Uma ponte que liga o Claude (ou Claude Code) ao editor do Elementor aberto no seu navegador. Você pede em português normal — "cria uma seção com 3 cards de depoimento" — e o Claude monta tudo direto no editor. Sem copiar, sem colar.

Antes de começar

Claude Code

Se ainda não tem, baixe em claude.com/claude-code

Extensão Atomik

Aquela que você já usa hoje pra editar Elementor no Chrome.

Editor Elementor

Uma página em modo edição aberta, pra ver as mudanças acontecendo.

01

Instale o Node.js

O Node é o "motor" que roda o MCP no seu computador. Você instala uma vez e esquece.

  • Acesse nodejs.org
  • Clique no botão verde "LTS" (Recommended For Most Users)
  • Abra o arquivo baixado e instale como qualquer programa
Conferir que deu certo: abra o Terminal (Mac) ou Prompt de Comando (Windows) e digite:
Terminal
node --version
Deve aparecer algo como v22.10.0. Apareceu um número? Tá pronto.
02

Instale o Atomik MCP

No mesmo Terminal/Prompt, cole este comando:

Terminal
npm install -g @atomik-dev/mcp

Demora 30–60 segundos. Vai aparecer várias linhas — é normal. Quando o cursor voltar a piscar, terminou.

Apareceu "permission denied" ou erro de permissão? No Mac/Linux, prefixe com sudo: sudo npm install -g @atomik-dev/mcp e digite sua senha.
03

Avise o Claude Code

Agora você precisa "ligar" o MCP ao Claude Code editando um arquivo de configuração.

No Terminal, cole:

Terminal
open -a "TextEdit" ~/.claude.json 2>/dev/null || (echo "{}" > ~/.claude.json && open -a "TextEdit" ~/.claude.json)

No Prompt de Comando, cole:

Prompt
notepad %USERPROFILE%\.claude.json

Se o Bloco de Notas perguntar pra criar o arquivo, clique "Sim".

No arquivo que abriu, cole isto (substituindo qualquer conteúdo existente — ou mesclando se já tiver outros MCPs configurados):

~/.claude.json
{
  "mcpServers": {
    "atomik": {
      "command": "atomik-mcp"
    }
  }
}

Salve o arquivo (Cmd+S no Mac, Ctrl+S no Windows) e feche.

Já tinha outros MCPs configurados? Adicione só a parte do "atomik" dentro do "mcpServers" existente — não substitua o arquivo todo.
04

Reinicie e teste

  • Feche o Claude Code completamente (Cmd+Q no Mac / fechar pela barra de tarefas no Windows)
  • Abra de novo
  • No Claude Code, digite na barra: /mcp
  • Deve aparecer "atomik" na lista com status conectado

Pra fechar o ciclo:

  • Abra o Chrome com a extensão Atomik
  • Vá pra qualquer página em edição no Elementor
  • Abra o painel lateral do Atomik
  • Procure o pontinho verde brilhando no topo — ele indica que a ponte está ativa

Funcionou. E agora?

Teste prático pra confirmar que está tudo ligado:

  • Editor do Elementor aberto, pontinho verde aceso no Atomik
  • No Claude Code, manda: "insere uma seção com um título e um botão primário"
  • Acompanha o canvas do editor — em segundos os componentes aparecem
  • Se aconteceu, você acabou de virar uma máquina de fazer página

Algo não funcionou?

Erros comuns e como resolver.

O comando npm install -g deu erro
Três possibilidades:

1. "command not found: npm" — o Node.js não foi instalado direito. Volta no passo 1, baixa de novo do site oficial e reinicia o computador.

2. "permission denied" / "EACCES" — no Mac/Linux prefixe com sudo: sudo npm install -g @atomik-dev/mcp

3. "404 Not Found" — confira se digitou exatamente @atomik-dev/mcp (com arroba e hífen).
O /mcp diz que o atomik está "failed" ou "disconnected"
Confira se o JSON do passo 3 está bem formatado:

1. Todas as chaves { } bem fechadas, vírgulas no lugar.
2. O "command" deve ser exatamente "atomik-mcp" (com hífen).
3. No terminal, teste se o comando existe: which atomik-mcp (Mac/Linux) ou where atomik-mcp (Windows). Tem que aparecer um caminho.

Se nenhuma das opções resolver, copie o conteúdo do ~/.claude.json pra jsonlint.com pra validar o formato.
O pontinho do Atomik fica vermelho ou cinza
O pontinho vermelho/cinza significa que o servidor MCP não está rodando. Razões:

1. O Claude Code está fechado — abra ele, o pontinho fica verde sozinho em até 10 segundos.
2. Outro Claude Code está rodando em background — feche todos, espere 10s e abre só um.
3. O servidor travou — no Mac/Linux: rm /tmp/atomik-mcp.lock e reabra o Claude Code.
O Claude diz "tool not found" quando peço algo do Atomik
O Claude Code não recarregou a configuração. Fecha o Claude Code totalmente (não só a janela — pela barra de tarefas / Cmd+Q) e abre de novo.
Quero atualizar pra uma versão nova
No Terminal/Prompt: 
npm update -g @atomik-dev/mcp
Depois reinicia o Claude Code. Pronto.
Quero desinstalar
No Terminal/Prompt: 
npm uninstall -g @atomik-dev/mcp
E remova a seção "atomik" do ~/.claude.json.
Nada funciona — onde peço socorro?
Abre uma issue em github.com/ciparoni/atomik-mcp/issues contando:

– Em qual passo travou
– Qual mensagem de erro apareceu (print ou texto)
– Qual sistema (Mac, Windows, Linux)
